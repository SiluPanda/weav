weav

The in-memory context graph database for AI

Graph + Vector + RAG in a single sub-millisecond Rust binary. Built for AI agents, GraphRAG, and LLM applications.

Get Started GitHub
Rust In-Memory MCP Native RESP3 / gRPC / HTTP

AI needs structured context, not just vectors

01

Vector databases return similar chunks but miss the relationships between facts. Similarity is not understanding.

02

Graph databases understand relationships but were never built for LLM token windows. They return too much or too little.

03

Every RAG pipeline stitches 3–5 tools together with brittle glue code. 40–60% of these pipelines never reach production.

Everything AI applications need. One binary.

One API call. Eight stages.

POST /v1/context
{
  "graph": "my-knowledge-base",
  "query": "How does Rust handle memory?",
  "embedding": [0.12, -0.03, ...],
  "seed_nodes": ["rust-lang", "memory-safety"],
  "budget": 4096,
  "max_depth": 2
}
  1. 01Resolve seeds via vector search (HNSW) and entity key lookup
  2. 02Score by graph flow relevance from seed nodes
  3. 03Fuse vector + BM25 + graph scores with Reciprocal Rank Fusion
  4. 04Build context chunks with content and metadata
  5. 05Apply temporal filter (point-in-time queries)
  6. 06Apply relevance decay (exponential, linear, or step)
  7. 07Pack context via greedy knapsack within token budget
  8. 08Return assembled context with provenance metadata

No other database offers a single-call, token-aware context assembly primitive.

Rust. Zero GC. Sub-millisecond.

  • Thread-per-core with keyspace sharding
  • Per-graph RwLocks for concurrent reads
  • String interning — labels stored as u16 IDs
  • SmallVec adjacency — stack-allocated for typical degrees
  • 11 modular crates, zero cyclic dependencies

Get started in 30 seconds

CLI / curl
# Start the server
./weav-server

# Create a graph
curl -X POST localhost:6382/v1/graphs \
  -d '{"name": "my-kb"}'

# Retrieve context
curl -X POST localhost:6382/v1/context \
  -d '{"graph": "my-kb",
       "seed_nodes": ["rust-lang"],
       "budget": 4096}'
Python SDK
from weav import WeavClient

client = WeavClient()
client.create_graph("my-kb")

ctx = client.context(
    graph="my-kb",
    embedding=embeddings,
    budget=8192
)

prompt = ctx.to_prompt()
